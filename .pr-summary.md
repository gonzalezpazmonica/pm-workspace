SE-228 Slice 1 introduces the canonical STATE.md schema and two companion scripts that give autonomous skills (overnight-sprint, code-improvement-loop, tech-research-agent) a persistent memory spine between sessions.

Before this change, every overnight-sprint run started from zero: previously-seen PRs were re-queued, ignored noise re-processed, and the history of attempts lost. The root cause was the absence of a standardised state file. SE-228 S1 fixes this with three deliverables.

The schema document (docs/rules/domain/loop-state-schema.md) defines four mandatory sections in every STATE.md: High Priority (items the loop is actively working or waiting on a human for), Watch List (monitored but not yet actioned), Recently Resolved (closed/merged items from the last 7 days), and Noise/Ignored (permanently discarded signals and items that exhausted their three attempts). Each High Priority entry carries an attempt counter (N/3) so the loop knows when to escalate to Noise rather than retry.

The init script (scripts/loop-state-init.sh) bootstraps output/loop-state/<skill>/STATE.md from the template with a single command. It is idempotent: a second call without --force exits 0 without touching the existing file, making it safe to call at the start of every run. --dry-run mode shows what would happen without writing anything.

The prune script (scripts/loop-state-prune.sh) walks the High Priority and Watch List sections, checks each referenced PR number against gh pr view and each branch name against git ls-remote, and moves resolved items to Recently Resolved with a timestamp and outcome. --max-resolved N (default 20) keeps the section from growing unbounded. --dry-run shows the planned moves without modifying the file.

The overnight-sprint SKILL.md gains a new Loop State section pointing to the schema and the init command, closing the discoverability gap for operators who read the skill before running it.

29 BATS tests cover both scripts end-to-end: creation, idempotency, force-overwrite, dry-run, empty-state graceful handling, max-resolved trimming, unknown-arg rejection, and schema document integrity. Test auditor score: 87/100 (certified, threshold 80).
