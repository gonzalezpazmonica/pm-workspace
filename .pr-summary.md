## Qué hace este PR (en lenguaje no técnico)

Savia tiene memoria de sesiones anteriores: guarda decisiones, patrones, bugs resueltos y contexto de proyectos en un fichero local. Esa memoria existe desde hace tiempo, pero cuando buscabas algo relevante ("¿qué decidimos sobre autenticación?") el sistema solo hacía búsqueda por palabras exactas (grep). Si recordabas la palabra equivocada, no encontrabas nada.

Este PR activa la búsqueda semántica que ya estaba construida pero nunca puesta en marcha: ahora Savia entiende el significado de lo que buscas, no solo las palabras literales. En un benchmark con 10 consultas representativas, la búsqueda por palabras encontraba el resultado correcto el 40% de las veces. Con búsqueda vectorial sube al 90%. La diferencia se nota especialmente en consultas como "problemas de autenticación" que antes no encontraban "Token refresh timeout".

El fallo silencioso era el problema real: el sistema degradaba a grep sin avisar, y no había forma de saber qué nivel de búsqueda estaba activo. Ahora existe `bash scripts/memory-store.sh doctor` que reporta exactamente el estado (Level 0/1/2), qué dependencias faltan, y cómo arreglarlo.

**Riesgo abierto**: `hnswlib` requiere compilar con `python3-dev` (no siempre disponible). El instalador usa `faiss-cpu` como alternativa precompilada que funciona sin sudo. En sistemas muy restringidos puede caer a Level 0, pero ya no silenciosamente.

**Cómo activar**: el instalador (`install.sh`) lo hace automáticamente. En instalaciones existentes: `pip install -r scripts/requirements-memory.txt && bash scripts/memory-store.sh rebuild-index`.
