## Qué hace este PR (en lenguaje no técnico)

Savia arrancaba cada sesión cargando ~22.000 tokens de información antes del primer mensaje. Este PR reduce eso casi a la mitad dividiendo el catálogo de 558 comandos en "siempre necesarios" (287) y "bajo demanda" (271).

Además resuelve un problema invisible: cada vez que el agente editaba un fichero, disparaba 44 procesos bash en paralelo (uno por hook registrado). Con el dispatcher que introduce este PR, esos 44 procesos pasan a ser 2 — uno que entra y uno que sale.

El resto es deuda técnica que se acumulaba: hooks registrados sin especificar qué herramienta los dispara, un fichero de 860 líneas de bash que procesaba JSON con herramientas de texto (ahora es Python de 887 líneas con tests propios), agentes duplicados entre dos directorios sin guardia que avisara cuando divergen, y specs implementadas pero que seguían apareciendo como "propuestas" en el backlog.
