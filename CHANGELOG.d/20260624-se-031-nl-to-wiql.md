## SE-031 — NL-to-WIQL/JQL query translator (2026-06-24)

### Added
- `scripts/nl-to-wiql.py`: pattern-based natural language to WIQL/JQL translator
  - `--query` / `--backend ado|jira` / `--json` flags
  - 9 ADO patterns: @CurrentIteration, @Me, Blocked, Active, Closed, Bug, PBI, Task, Priority
  - 8 JQL patterns: openSprints(), currentUser(), status, issuetype, priority
  - Combined pattern matching (mis items sprint actual → both @Me + @CurrentIteration)
  - Mock JSON result scaffold with items array
- `tests/scripts/test_nl_to_wiql.py`: 22 pytest tests covering:
  - "sprint actual" → @CurrentIteration (ADO + JQL)
  - "mis items" → @Me / currentUser()
  - "bloqueados" → State="Blocked"
  - backend ado vs jira produce syntaxis distinta
  - output JSON válido con campos requeridos
  - query vacío → error claro (exit 1)
  - combined patterns (mis items sprint actual → both filters)

### Design notes
- Pure pattern library: deterministic, no LLM, no network
- Aligns with SE-031 Slice 1 scope (resolver + NL heuristic)
- Extends existing `query-lib-nl.sh` pattern with Python-native solution
