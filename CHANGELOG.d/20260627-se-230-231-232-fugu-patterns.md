## [Unreleased] — 2026-06-27 · SE-230/231/232 Fugu Loop Patterns — PR #883

Added: three specs inspired by Sakana Fugu architecture (arXiv:2606.21228v2) that shift
Savia loops from administratively bounded execution to convergence toward verifiable objective.

- SE-230 Auto-Loop Gate (`scripts/auto-loop-gate.sh`): classifies incoming requests into
  PROPOSE_LOOP / SINGLE_SHOT / CLARIFY_NEEDED based on verifiable DoD detection. Decision
  table maps 6 request patterns to loop skills with convergence criteria. Opt-in via
  `SAVIA_AUTO_LOOP=enabled`. Integrated into `user-prompt-intercept.sh`.
- SPEC-RECURSION-GUARD: `recursion-guard.sh` PreToolUse hook blocks loop-in-loop recursion
  via `SAVIA_LOOP_CONTEXT` env variable. `recursion-guard-export.sh` sources at loop startup
  to export depth-tracked context (format: `loop_name:depth`).
- SPEC-TERMINAL-STATE-HANDOFF: `terminal-state-emit.sh` / `terminal-state-read.sh` provide
  structured termination with 6-state enum (completed / user_abort / token_budget / stop_hook /
  max_turns / unrecoverable_error) and differentiated exit codes for precise retry policies.
- SE-231 Adaptive Turn Routing (`scripts/court-turn-router.sh`): routes Code Review Court
  judges based on dominant finding type after each fix round. Security → {security-judge,
  correctness-judge}. Architecture → {architecture-judge, spec-judge}. Mixed/last → all 5.
  Expected ~40% cost reduction on rounds 2–3.
- SE-232 Workflow-as-Output (`scripts/dev-workflow-generate.sh`): generates adaptive YAML DAG
  from a spec file instead of static slice template. Detects security implications, language,
  test requirements, and parallel opportunities.
- Fixed: `recursion-guard.sh` hook path in `.claude/settings.json` was missing
  `$CLAUDE_PROJECT_DIR` — hook was silently never executed.

Tests: 107 BATS across 5 suites (test-recursion-guard, test-court-turn-router,
test-terminal-state, test-se-230-auto-loop-gate, test-dev-workflow-generate). All passing.
