## Reglas para futuras Eras

Una Era nueva se justifica cuando concurren **≥3 de estos criterios**:

1. ≥3 PRs estructurales fusionados con un tema cohesionado
2. Cambio en la arquitectura de contexto, modelo base, o framework de seguridad
3. Introducción de un nuevo lenguaje de dominio o patrón de diseño en el workspace
4. Cierre o migración de una deuda técnica mayor (≥5 specs APPLIED)

**Para añadir una Era**: PR con `[era-add]` en el commit message y entrada en esta tabla.

> Nota: las Eras con descripción `(referenciada; sin descripción recuperable)` tienen evidencia de existencia en docs/ pero ningún documento canónico que defina su contenido con certeza. Si tienes esa información, envía un PR con `[era-backfill]` en el commit message.
